# Code Of Conduct

## Purpose

This project should remain a practical, respectful place to collaborate on YubiKey tooling and documentation.

## Expected Behavior

- Be respectful and constructive.
- Focus criticism on technical behavior, security properties, documentation clarity, and maintainability.
- Assume security reports may involve sensitive details and handle them carefully.
- Respect that contributors may have different levels of experience with YubiKeys, OpenSSH, Bash, and GitHub.
- Help keep public issues and pull requests free of secrets, private key material, serial numbers, credential IDs, and personal command output.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Publishing someone else's private information or sensitive security material.
- Pressuring users to disclose PINs, passphrases, private keys, recovery codes, management keys, or account-linked key material.
- Deliberately derailing technical discussion or repeatedly ignoring maintainer guidance.
- Posting exploit details publicly when private vulnerability reporting is appropriate.

## Security-Sensitive Discussion

If a report could help someone attack users, expose credentials, or misuse YubiKey metadata, do not discuss details in a public issue.

Use GitHub private vulnerability reporting:

https://github.com/pal-ayan/yubikey/security/advisories/new

## Enforcement

Project maintainers may edit, hide, or remove comments, issues, pull requests, or other contributions that violate this Code of Conduct. Maintainers may also block users from participating when behavior creates risk for contributors, users, or the project.

Enforcement decisions should be proportionate to the issue and focused on keeping the project safe, useful, and respectful.

## Scope

This Code of Conduct applies to participation in this repository, including issues, pull requests, discussions, documentation, code review, and security-reporting interactions.
